- **Root cause:** `memref_ids()` in `agent-skills/memory-lint/scripts/memory-lint.py` located the
  references section with an **un-anchored** `text.find("## Memory References")`, so it matched the
  *first* occurrence anywhere in a file ‚Äî including the heading string quoted inline in a session
  log's prose. The review log quoted it while describing the check, and lint then read the log's
  `## What happened` paragraph as the references block, scooping the archived id in ‚Üí false
  `[over-archived] sync-adapters-v420`.
- **Fix:** anchor the heading to a real line (`(?m)^## +Memory References[ \t]*$`) and bound the
  block at the next line-anchored heading (`(?m)^## +\S`). Prose that *quotes* the heading (not at
  line start) is now ignored.
- **Verified** with three added cases run against the patched function: (1) inline-quoted heading +
  a real `(none)` references section ‚áí id NOT counted (false-positive gone); (2) genuine references
  in the real section ‚áí still detected; (3) bounding still stops at the next heading. Full repo run:
  `OK: 0 errors, 0 warnings`.
