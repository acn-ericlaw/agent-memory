**Also in this session: v4.16.1 (PATCH)** ‚Äî session filename drift fix.

Root cause: `templates/AGENTS.md` said "Use `date -u` *or equivalent*", allowing agents to
treat the context-injected `currentDate` (date-only) as equivalent; `memory-lint` had no
filename check so drift was silent. Surfaced when this session wrote `2026-06-23.md` instead
of `2026-06-23-153401.md`.

Three-layer fix:
- `templates/AGENTS.md` step 1 + checklist: explicit "always run `date -u +%Y-%m-%d-%H%M%S`";
  names the `currentDate` trap
- `templates/.agent/schema.md`: same prohibition
- `memory-lint.py` + `memory-lint.mjs`: `check_session_filenames` (check 5) flags `YYYY-MM-DD.md`
  sessions as `[date-only-session]` warnings; exported from Node module; 3 tests in each suite
  (11/11 pass Python, 11/11 pass Node)

Running the linter against this repo now emits the expected warning for the pre-existing
`memory/sessions/2026-06-12.md` (a pre-protocol multi-session file with no timestamp; no
fabricated rename ‚Äî the warning is correct and intentional).
