scan misses. Safe because it writes **only gitignored** files (no committed change, no version bump,
no session log per the lightweight-mode rule) and `no-build-step-agent-run` holds — the agent runs it
during a human-invoked enable/upgrade, not a daemon. Per-session path unchanged (still never touches

## Decisions Made

- **Enforce, not recommend, at enable + upgrade.** These are deliberate human-invoked moments; the
  right outcome is materialized adapters, not advice the user must act on before skills work natively.
- **Unconditional sync over a drift-gated run.** Idempotent + gitignored-only makes "always run" the
  simplest correct design, and it also catches content drift a filename-only scan can't.
- **MINOR (4.12.0).** Additive, backward-compatible behavior change; no memory-file shape change; an
  un-upgraded agent still works.
- **Preserve the per-session lightweight principle.** Only enable/upgrade gained the sync; the
  session ritual still never touches skills, and a gitignored-only sync is explicitly no-log.
