## Summary

Shipped **v4.23.2 (PATCH)** — **context-hygiene guidance**: teach *when* to compact a long AI session.

**The signal:** the maintainer, running multi-hour sessions, noticed context accumulation behaves like
human "brain fog" — a filling window degrades retrieval (specifics blur, decisions conflate) — and asked
when the best time to compact is. We worked out the principle, then decided to encode it as a best practice
so users inherit it.

**The principle (now in `AGENTS.md`):** compact/summarize a long session **at a natural seam** (milestone,
phase shift, before a topic pivot), **never mid-task** with hot unwritten state, and **always after** writing
the session log + `continuity.md`. Externalize first → the durable layer is the source of truth and the
summary sheds only volatile scratch (compaction keeps the narrative, not the verbatim texture). The session
log already **is** the seam marker: "when do I compact?" == "when do I write the log?". Reinforces *why* the
memory layer lives in files, not the chat buffer.

**Revised in-session (maintainer review):** two corrections to the first draft — (1) the agent usually
**can't compact itself** (compaction is user-invoked or a harness auto-compact), so the block now centers
the agent's real lever: *externalize state*, not "self-compact"; (2) "retrieval degrades over hours" is a
judgment call — the **objective** health signal is **context-window utilization (tokens vs. limit)**, which
the harness tracks, while time and a felt "fog" are only proxies. Block retitled "Keep state externalized so
compaction is safe"; CHANGELOG/README/UPGRADE reframed to match.
