3. if yes, **do the deep analysis instead** and **record that decision in the first session log**.

**Implemented (operator-side — `ENABLE.md` only):**
- **Mode A (Step 3)** now opens with an advisory message (AI-enable *adds* a vendor-neutral
  `memory/` layer; it does **not** replace the CLI's own setup) and offers a **discovery depth**:

    control/data flow, conventions, build/test/CI) *plus* the markdown harvest.
- **Key reframe (mine):** deep analysis borrows the *capability* of a vendor's `/init` but writes
  findings into the **neutral** memory layer (`instructions.md` + `continuity.md`), **never** a
  vendor steering file — *capability, not destination*. And deep **subsumes** (doesn't skip) the
  4b harvest, so docs are never dropped (avoids re-introducing the v4.22.0 complaint).
