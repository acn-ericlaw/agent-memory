## Decisions Made

- **No dispatcher / no installer / no `detect_runtime.sh`.** The agent is the runtime
  (`no-build-step-agent-run`); a sourced shell wrapper would break native-Windows/Kiro and add a
  bootstrap step. `SKILL.md` documenting both commands is the vendor- and OS-neutral dispatch. (This
  was the conclusion of evaluating a fuller "polyglot runtime plan" draft — rejected as over-engineered
  for a one-script population; kept minimal per the user's guiding principle.)
- **Parity is enforced by the test contract, not by hope.** Two implementations of the safety-critical
  decay logic only stay trustworthy if the same fixtures gate both — so the node tests are an exact
  mirror, and the determinism guarantee is what's being protected.
- **MINOR, not PATCH.** A new runtime is additive new capability (backward compatible) → 4.11.0, matching
  how `memory-lint` itself (4.9.0) and other capabilities were versioned.
- **No runtime-free hand-count fallback — install-or-defer instead.** The decay guarantee is that a
  committed archival decision is *always* script-verified; if neither runtime is present, halt the
  decision (and advise a one-time install) rather than estimate it. Keeps `no-build-step-agent-run`
  intact (the *layer* still needs no runtime; only *committing a decay decision* does). Maintainer's
  framing: a skilled dev almost always has one runtime, and the realistic "neither" case is a thin CI
  container — where a hard stop is the right behavior, not a degradation.
