## What We Did

Designed and shipped **P1 as v3.3.0**: the evolving-memory layer can now represent a
fact becoming *false* (a decision reversed, a dependency dropped), not just fading from
disuse — the industry-alignment assessment's one "High" gap. Confirmed the design with
the user (one fork: mark superseded **immediately** vs. defer to review → chose
immediate, treating it as a truth-state edit the agent owns under manual-override-wins).

The model: a terminal **`superseded`** tier + optional **`superseded-by`** /
**`supersedes`** footer fields. On a reversal the agent adds the successor (born
`tier: working`, `supersedes: <old>`), marks the old fact `tier: superseded` +
`superseded-by: <new>` immediately, and records `Superseded: <old> → <new>` in the
session log's `## Memory References`. The review archives superseded facts flagged
"superseded" (distinct from "faded"), promptly (no `archive_window` wait), links
preserved; terminal — never reactivated by a reference (only a human can reverse it).
It's the markdown-native analogue of bi-temporal `valid_at`/`expired_at`/`invalid_at`,
with no scoring.

## Decisions Made

- Supersession is **immediate** (truth-state edit the agent owns), not deferred to the
  review — a false fact shouldn't linger. Recorded as an event for the ledger too.
- **MINOR → 3.3.0**: additive (optional fields + a new tier), backward-compatible
  (repos without superseded facts unaffected; a 3.2 agent following "manual override
  wins" leaves superseded facts alone). Added an `UPGRADE.md` 3.2.0→3.3.0 rung.
- Pure invalidation (no successor) = `tier: superseded` with no `superseded-by`.
