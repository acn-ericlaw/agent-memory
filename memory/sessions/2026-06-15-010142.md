## What We Did

Hardened the design to be **process-neutral**: an enabled target's owner may layer SDLC /
scrum on top, or not — and `memory/` must survive either way without getting heavy or
absorbing ceremony. Encoded the principle in two homes:

- **`DECAY.md` §12** (ships into every target): a "process-neutral — survives whatever the
  target chooses" rule. Ceremony + scoring (velocity, story points, estimates) stay in the
  target's own space, **never in `memory/`** (which stays determinism-pure, no float); the
  loop is **cadence-agnostic** (a sprint boundary = "run a review"); extra tags like
  `(sprint)` coexist with the primitives.
- **`docs/DESIGN-vbdi-lifecycle.md` §13** (rationale): neither require nor forbid SDLC;
  reuse the existing primitives (backlog = Blueprint, sprint = a bounded gap subset, DoD =
  smoke test + altitude trace, retro = review, PO/SM = the human gates); the lightweight
  non-goal constrains the *tool*, not the target's freedom.

Added `bp-sdlc-overlay` as an **optional** Blueprint gap (a future scrum profile for
targets — `(sprint)` tag + sprint-boundary review, no points/ceremony; only if a real
target wants it).

## Decisions Made

- The memory substrate is **process-neutral by design** — it bends to the target's chosen
  process weight without the core getting heavy or the substrate absorbing ceremony.
- The failure mode to guard against is scoring/ceremony leaking into `memory/`; guarded by
  the no-float principle + the explicit §12/§13 rule.
- **No version bump** — design clarification/hardening; `VERSION` stays 4.0.0 (propagates
  to enabled repos on their next `DECAY.md` re-sync).
